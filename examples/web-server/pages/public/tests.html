Status

Caneka is a runtime and set of programming langauge tools. Currently targeted for interactive web and email software development and products.

How it works

Built on the basics of memory allocation, data structures, and network serving, Caneka is a fresh start for interactive software development.

The system allocates 16 megabytes at a time, then assigns a page at a time (4k) to chapters that reside in the books (16mb sections). These blocks of memory are owned by tasks. The tasks use basic data structures to form parsers, queues, and the infrastucture for parsing, transforming, and serving interactive content.

The system builds with only one optional dependency (NaCL for cryptography) and builds using a bootstrap file (also written in C). It only relies on the POSIX (Linux/Mac/Windows/*BSD) standard, no operating system specific code is contained in the system. This makes the system very independent.

What's Done/Whats left

All foundational components are done, and the login server/email server is now bing assembled. After that, interactive browser based application development beings!

From a time elapsed perspective the system feature list is unbalanced.

This is good news, because the core components are accelerating development of the newer more advanced features.

Core components have taken 21 months, advanced components such as Dynamic Types, BinSeg, WebServer, and a new Build System happend in the last three months (Sept-Nov 2025).
